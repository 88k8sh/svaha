Install

Four steps, ~2 minutes. It needs claude, python3, and git on your machine.

1Terminal  Check the three prerequisites — each should print a version, not "command not found":
claude --version && python3 --version && git --version
2Terminal  Clone it and move in. Put it somewhere permanent (e.g. ~/svaha) — the hooks reference this folder by absolute path, so moving it later stops the guards firing:
git clone https://github.com/88k8sh/svaha && cd svaha
No git? Download the repo as a ZIP, unzip, and cd into the folder instead.
3Terminal  Run the installer. It copies the 7 slash commands + 6 hooks into ~/.claude/ and wires your settings.json for you — filling in the paths, validating the JSON, and (if you already have a settings.json) safely merging into it with a backup instead of overwriting. Just answer its prompts — return takes the safe default. Non-destructive; it asks before touching an existing CLAUDE.md:
./setup.sh
At "Generate it now…? [Y/n]" (that's the hooks + permissions) press return for yes. If you already have a settings.json, it offers a safe merge instead — your other keys stay untouched and a timestamped backup is kept.

Choose your edit mode (optional)

The snippet ships frictionless-but-fenced: editing is granted once at install, reads run silent, and git push / rm / network / secret-reads always confirm or block. Prefer to be asked once per session before edits? Delete the three lines "Edit(**)", "Write(**)", "MultiEdit(**)" from the "allow" array.

Wire settings.json by hand (only if you skipped it, or have no Python)

setup.sh already did this. If you answered no, re-run ./setup.sh and press return at the settings prompt. To do it yourself from inside the svaha folder — fresh write (no existing settings.json):

mkdir -p ~/.claude && sed '/^[[:space:]]*\/\//d' settings.json.snippet | sed "s||$(pwd)|g" > ~/.claude/settings.json && python3 -m json.tool ~/.claude/settings.json >/dev/null && echo "OK — settings.json written and valid"

Already have a settings.json? Don't overwrite it. Open settings.json.snippet, replace every <kit-dir> with this folder's path (pwd), and merge its "permissions" and "hooks" blocks into yours — per-array, not whole-object. Then validate with python3 -m json.tool ~/.claude/settings.json.

4Terminal  Confirm it landed. The doctor checks the commands, hooks, and settings.json (valid JSON, paths wired, no leftover placeholder) and names the exact fix for anything off:
bash bin/doctor.sh
ends with ✓ All good — Svaha is wired up. Any ✗ line above it tells you the one thing to fix, then just re-run the doctor.

Your first ten minutes

The fastest way to feel the loop is to run it once, end to end. One word drives the whole thing — svaha ("so be it / offered into the fire"); where you svaha decides what it does.

  1. Boot it. In a project, start Claude Code and just type svaha — it opens the launchpad and resumes where you left off. A brand-new project has nothing to resume yet; that's expected. (Explicit: /session.)
  2. Do a small task — fix a bug, add a function. Notice it states decisions with conviction and acts on the safe stuff without nagging, but stops to confirm anything risky.
  3. Wrap up. When you're done (or context is filling), type svaha again — or "that's it." It freezes a numbered _NEXT capturing where you stopped and what you decided, then seals the session. (Explicit: /handoff.)
  4. Start fresh. Open a new session and svaha. This time it recovers where you left off — and reconciles its claims against the actual files before doing anything.
  5. That's the loop. Each pass is a little better calibrated than the last. The longer you run it, the more it knows.

You type svaha (or any clear yes — amen works too); when a wrap completes it says the one seal back, so you know it landed. Every session ends one of two ways — you'll see the matching banner at the very foot of the chat, the same as here:

स्वाहा The firm close — your work is sealed into a numbered handoff and recorded. The thread is saved; the natural next move is a fresh session.
◇ ◇ ◇ The soft close — a light session with nothing to carry. Nothing forces a stop: keep going, or set it down with a clean slate.

Not sure it's wired up?

Run the doctor any time — it checks commands, hooks, settings, and paths, and tells you exactly what to fix:

You rarely type a slash command — the keyword covers it — but each one maps to a command you can reach for directly.

Where things live

Make it yours

It's modular. CLAUDE.md splits along the risk gate: a core (safety skeleton + handoff loop) and a clearly-fenced operator layer (the opinionated judgment half — conviction calibration, premortem, weigh-before-you-ask). Keep it for the full self-tuning experience, or delete that one block for a lean core that still stops on everything dangerous.

python3: command not foundInstall Python 3 — the guard hooks are Python.
A guard can't find a bin/ scriptThe svaha folder moved. Move it back, or re-run ./setup.sh (or the manual fallback under Step 3) so <kit-dir> points at its new path.
/session says "nothing to resume"Expected on a fresh project — no prior handoff yet. Do some work, /handoff, and the next /session recovers it.
