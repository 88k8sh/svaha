The cost was never forgetting.
It's re-establishing — every single time.

Models remember fine — that was never the problem. The tax is everything around it: every resume, you paste the task back in and hunt for where you left off — and the moment you run more than one session, they clobber each other.

I built it to stop paying that tax — what I didn't expect was how much steadier the work got once the thread lived in the system instead of my head. Seeing स्वाहा come back is the all-clear: written to file, safe to close.

the spineThe handoff

Each session ends by writing one numbered file, never edited again — a correction becomes the next file, the way a commit is only ever followed by another. A fresh session reads the latest open one and resumes mid-sentence. Run several at once and each takes its own slot, so parallel work never collides — no session ever again opens with so, what were we doing?

it's differentWhy

It's one word

svaha opens, confirms, and closes — no command palette, no syntax. The simplest possible surface over everything running underneath.

It's mine, and I can read it

Plain files in my own repo. When the assistant has something wrong, I open the file and fix it — nothing parked silently where I'll never see it.

It doesn't repeat mistakes

When something breaks, the cause gets traced and a guard gets written — a rule, a check, a hook — so that whole class can't recur. Every failure hardens the system.

It's portable

Context lives in my system, not a provider's. It comes with me when I switch tools or models — never locked in a memory I can't carry out.

It's future-proof

Plain files and a few hooks — no database, no lock-in. The day a model ships this natively, or you outgrow it, you delete a folder and walk away.

It stays lean

Each session opens from a small checkpoint, not the whole history — the window never bloats, and you don't pay to re-load everything. Fast and cheap, every time.

It compounds

Most setups degrade as they accumulate — more drift, more stale instructions. The guard layer is why this one doesn't: worth more after six months, not a liability.
