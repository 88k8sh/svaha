Install

Svaha — Install

From scratch, click-to-copy. Every command has a Copy button — paste it into a Terminal, press return.

Open this file in any browser. Works offline (the brand fonts fetch when you're online).

Terminal = a shell window — paste & run.   Edit file = open a file in a text editor.   Claude = type into a running Claude Code session.

0 · Prerequisites

Never used Claude Code before?

This page installs Svaha on top of Claude Code — it assumes claude already runs. If you're starting from a fresh Mac, do INSTALL-CLAUDE-CODE.html first (in this same folder): it installs Claude Code, logs you in, and sets up git + python3 so the check below comes up all green. Then come back here.

Terminal Check you have the three things Svaha needs. Each should print a version, not "command not found":

claude --version && python3 --version && git --version
If claude is missing → install Claude Code first (see INSTALL-CLAUDE-CODE.html). If python3 is missing → install Python 3 (the guard hooks need it). git is only needed for the clone in Step 1 — you can download the folder instead.

1 · Get Svaha

1Terminal  Clone it and move into the folder:
git clone https://github.com/88k8sh/svaha && cd svaha
No git? Download the repo as a ZIP, unzip it, then cd into the unzipped folder instead.

Keep this folder where it is

The hooks reference this folder's bin/ scripts by absolute path. Put it somewhere permanent (e.g. ~/svaha) before Step 3 — if you move it later, the guards stop firing.

2 · Run the installer

2Terminal  Copies the 6 slash commands + the hook scripts into ~/.claude/. Non-destructive — it asks before touching an existing CLAUDE.md. Answer its prompts:
./setup.sh

3 · Wire the hooks + permissions (the one fiddly step — made easy)

This merges settings.json.snippet into your Claude config, with this folder's path filled in for <system-dir>. Pick the path that matches you:

3A · If you do NOT already have a ~/.claude/settings.json (most new installs)

3ATerminal  Run this from inside the svaha folder. It strips the comments, fills in <system-dir> with your real path, writes the file, and checks it's valid — one paste:
mkdir -p ~/.claude && sed '/^[[:space:]]*\/\//d' settings.json.snippet | sed "s||$(pwd)|g" > ~/.claude/settings.json && python3 -m json.tool ~/.claude/settings.json >/dev/null && echo "OK — settings.json written and valid"
prints OK — settings.json written and valid. If it prints a JSON error instead, nothing was broken — re-run from the correct folder.

3B · If you ALREADY have a ~/.claude/settings.json

3BEdit file  Open settings.json.snippet and your existing ~/.claude/settings.json, then:

1. In the snippet, replace every <system-dir> with this folder's full path. Get it with:

pwd

2. Merge the snippet's "permissions" and "hooks" blocks into your settings.json (if you already have those keys, merge per-array / per-event — don't replace the whole object). 3. Validate:

python3 -m json.tool ~/.claude/settings.json >/dev/null && echo "valid JSON"

Choose your edit mode (optional)

The snippet ships Mode A: editing is granted once at install, so Claude edits files without re-asking. Reads/inspection are always silent; git push, rm, network, and secret-reads are always gated. Prefer to be asked once per session before edits? Open ~/.claude/settings.json and delete the three lines "Edit(**)", "Write(**)", "MultiEdit(**)" from the "allow" array. The snippet's header comment explains both.

4 · Boot it

4Terminal  Go to any project (or stay here) and start Claude Code:
claude
Claude  Boot the system — just say svaha (or run the explicit /session):
/session
It boots and prints a proof-of-boot line listing the files it loaded. A brand-new project has nothing to resume yet — that's correct.
Claude  That's the whole loop: svaha to begin, svaha (or "that's it") to wrap when you're done — it freezes the handoff and seals with स्वाहा, the receipt that your work is recorded. (The slash commands /session & /handoff are the explicit equivalents.) WELCOME.html walks the loop end-to-end.

5 · Confirm it worked

5Terminal  The 6 commands and 6 hooks should be in place:
ls ~/.claude/commands && echo "---" && ls ~/.claude/hooks
commands: boot.md session.md handoff.md reflect.md audit.md foldin.md hooks: the six *.sh files (context-canary, launchpad-nudge, memory-reflect, resume-line-guard, session-end-backstop, session-start-marker)
Claude  And let the system check its own coherence:
/audit

Just want the behavior, not the full loop?

Skip the scaffold — drop the contract in and you're done (no handoff loop, no hooks):

cp CLAUDE.md ~/.claude/CLAUDE.md

Global for every project, or use cp CLAUDE.md ./CLAUDE.md to scope it to one repo.

6 · If something snags

SymptomFix
Hooks never fire / commands behave oddlyYour settings.json is probably invalid JSON or still has a literal <system-dir>. Re-run the validate: python3 -m json.tool ~/.claude/settings.json — fix what it flags.
python3: command not foundInstall Python 3 — the guard hooks (security-guard.py, etc.) are Python.
A guard complains it can't find a bin/ scriptThe svaha folder moved. Either move it back, or re-do Step 3 so <system-dir> points at its new path.
/session says "nothing to resume"Expected on a fresh project — there's no prior handoff yet. Do some work, then /handoff, and the next /session will recover it.
command pasted weirdly / "command not found"Use the Copy button (it copies the exact text) — don't hand-select, which can grab stray characters.
